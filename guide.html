이벤트 색상

수업랩시험과외워크숍마감

4 공부 플랜 탭

6/4(목) ~ 6/15(월) 12일간 기말고사 준비 플랜. 3단계로 구성돼요:

Phase 1 (6/4–6/9) — 기초회로 집중
Phase 2 (6/10–6/12) — 기초회로 마무리 + 공학수학 병행
Phase 3 (6/13–6/15) — 시험 주간

각 날짜 카드에 추천 공부 블록·시간·주의사항이 표시돼요. 과외·수업 일정을 고려해 실제 가능한 시간대만 표시한 거예요.

5 충돌 알림 탭

달력 데이터를 자동 분석해서 문제 구간을 감지해요. 새 일정이나 이동시간을 추가하면 실시간으로 재계산돼요.

충돌 이동시간 초과
이동에 걸리는 시간이 과외 사이 여유보다 길어요. 즉시 조정 필요.
빠듯 여유 30분 미만
이동은 가능하지만 여유가 없어요. 지각 위험.
⚠ 잠실 파크리오 관련 구간은 이동시간 미등록 상태예요. 편집 탭 → 이동시간 추가에서 등록하면 충돌 분석에 반영돼요.

6 편집 탭 — 핵심 기능

① 자연어로 추가 (Gemini AI)

텍스트박스에 자유롭게 입력 → AI 파싱 버튼 → 미리보기 확인 → 추가 버튼.

이벤트 추가:
정현우 6/28 토 17시-20시 잠실 파크리오 대면 과외
이민희 최지요 6/25 목 19-21시 온라인
신수민 7/5 토요일 9:30-12:30 스카이캐슬 물리
이동시간 추가:
잠실 파크리오 → 강남역 옐로스톤 25분 지하철
서울대 관악 → 잠실 파크리오 40분

AI가 이벤트인지 이동시간인지 자동으로 판단해요. 확인 후 저장 누르면 GitHub에 즉시 반영.

② 언제 잡을까? (AI 추천)

현재 달력 전체를 Gemini에게 보내서 빈 시간대를 찾아 추천받는 기능이에요.

정현우 과외 대면 2시간, 이번 주 일요일 가능한 시간 추천해줘
다음 주 중 공부할 시간 3시간 확보하고 싶어
6월 셋째 주에 신수민 과외 넣고 싶은데 언제 괜찮아?

추천 결과는 텍스트로 보여줘요. 마음에 들면 위의 "자연어로 추가"에 그대로 복붙해서 저장하면 돼요.

③ 이벤트 직접 추가

날짜·타입·이름·시간·장소를 폼으로 직접 입력하고 저장. AI 없이도 사용 가능해요.

④ 이동시간 추가/수정

출발·도착 장소 선택 → 소요시간(분) 입력 → 교통수단 선택 → 저장. 양방향 자동 등록이에요.

⚠ 현재 미등록 구간: 잠실 파크리오 ↔ 강남역 옐로스톤 / 서울대 ↔ 잠실 파크리오

⑤ 설정

GitHub PAT — 저장 권한. github.com/settings/tokens에서 repo 권한으로 발급.
Gemini API Key — AI 기능. aistudio.google.com에서 무료 발급.
GitHub Repo — 기본값 schedule. 저장소 이름 바꿨을 때만 수정.

⚠ 키는 브라우저(localStorage)에만 저장됨. 기기마다 한 번씩 입력 필요.

7 앱 기능 업데이트 방법 (데이터 보존)

앱에 새 기능을 추가할 때 일정 데이터(events.json)가 날아가지 않는 방법이에요.

💡 원리: 일정 데이터는 GitHub의 events.json에 저장돼요. HTML(index.html)을 업데이트해도 데이터 파일은 건드리지 않아요.

Claude에게 기능 추가 요청 후 업데이트 순서

1
Claude가 index.html 수정 완료 후 PowerShell 열기
2
반드시 pull 먼저 — 앱에서 추가한 최신 데이터를 로컬로 가져옴
cd C:\Users\ASL\Desktop\Projects\schedule
git pull
3
index.html만 add — 데이터 파일은 건드리지 않음
git add index.html
git commit -m "기능 추가: 내용 요약"
git push
4
1~2분 후 사이트 새로고침하면 업데이트 반영
⚠ 절대 git add . 하지 마세요! 로컬의 events.json(초기 버전)이 GitHub의 최신 데이터를 덮어써요.

9 문제 해결

⬤ 동기화 점이 회색이에요
로컬 파일로 열었거나 네트워크 없음. GitHub Pages URL로 접속하면 초록색으로 바뀌어요.
AI 파싱 오류 / "Gemini 오류"
설정에서 Gemini API 키 확인. 키가 만료됐으면 aistudio.google.com에서 새로 발급.
GitHub 저장 실패 ("HTTP 401")
GitHub PAT가 만료됐어요. github.com/settings/tokens에서 새 토큰 발급 후 설정 업데이트.
사이트에 업데이트가 반영 안 돼요
GitHub Pages 배포에 1~3분 걸려요. 강력 새로고침(Ctrl+Shift+R / Cmd+Shift+R) 해보세요.
일정을 잘못 추가했어요
github.com/jdyece25-byte/schedule → events.json → 연필 아이콘 → 해당 항목 삭제 → Commit changes.
